# ARCHITECTURE.md — MebelLegal KZ

## Архитектурный стиль

**Модульный монолит** на Next.js 16 (App Router) + Supabase PostgreSQL.

Микросервисы, отдельные очереди, второй backend и дополнительные хранилища добавляются только после доказанной необходимости.

## Принципы

1. **Tenant isolation** — каждая пользовательская таблица имеет `organization_id` и RLS.
2. **Доменные команды** — изменения через проверяемые команды, не через произвольные запросы.
3. **Append-only audit** — юридически значимая история не перезаписывается.
4. **Версионность** — подтверждённые данные хранятся как версии, не перезаписываются.
5. **Идемпотентность** — повтор команды не создаёт дубль.
6. **Деньги в тийинах** — bigint, не float/number.

## Структура модулей

```
src/modules/
├── identity/          # Аутентификация, сессии
├── organizations/     # Организации, участники, роли
├── cases/             # Юридические кейсы, state machine
├── audit/             # Append-only audit log
└── shared/            # Money, ошибки, утилиты
```

## Будущие модули (границы зафиксированы, код не реализован)

- `legal-sources` — реестр правовых источников ИПС «Әділет»;
- `rules` — правила юридической проверки;
- `templates` — версии шаблонов документов;
- `documents` — генерация договорного комплекта;
- `consent` — согласование клиентом;
- `change-orders` — изменения заказа;
- `claims` — претензионный контур;
- `integrations` — связь с MebelDocs AI и Interactive KP.

## Поток данных (целевой)

```
источник заказа
→ юридический кейс
→ проверка данных
→ проверка правилами
→ версия договорного пакета
→ ручное утверждение
→ согласование клиентом
→ неизменяемая история
→ передача утверждённых данных
```

## Границы трёх систем

| Система | Домен |
|---|---|
| **MebelLegal KZ** | Правовые источники, правила, шаблоны, кейсы, договоры, согласования, audit log |
| **Interactive KP** | Коммерческие предложения, варианты комплектации, маркетинговые статусы |
| **MebelDocs AI** | Счета, платежи, бухгалтерские документы, ЭСФ, банковская сверка |

**Запрещён прямой доступ к таблицам соседней системы.** Связь только через версионированные API-контракты.

| База данных | Supabase PostgreSQL |

| Авторизация | Row Level Security (RLS) |

| Тесты | Vitest |

## Среды

| Среда | Назначение |
|---|---|
| `demo` | Только синтетические данные, внешние действия заблокированы |
| `pilot` | Ограниченные реальные данные утверждённых компаний |
| `staging` | Тест релиза и миграций |
| `production` | Только после security/compliance gates |

**Нельзя использовать production как тестовую среду.**
