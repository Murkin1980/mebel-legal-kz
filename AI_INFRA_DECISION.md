# AI_INFRA_DECISION.md — MebelLegal KZ

## Решения по AI на этапе 1

### Статус: AI отсутствует

На первом этапе (0–1) AI/LLM функционал **не реализуется**.

Причины:

1. Фундамент проекта должен быть создан до подключения AI.
2. Мультитенантность, RLS и доменные команды — приоритет.
3. AI добавляет сложность, которая не оправдана на этапе ядра.

### Запрещённые использования AI (этап 1)

- суммы и проценты;
- сроки;
- прав доступа;
- state transitions;
- выбор применимой редакции без детерминированной проверки даты;
- публикация нормы, правила или шаблона;
- подписание и отправка;
- признание или отклонение претензии;
- изменение подтверждённого пакета;
- удаление истории.

### Будущий контур AI

```text
AI proposal
→ источник и confidence
→ проверка пользователем
→ доменная команда
→ audit event
```

### Разрешённые использования AI (будущее)

- извлечение полей из документов;
- классификация;
- поиск по утверждённому корпусу;
- объяснение результата правила;
- сравнение редакций;
- подготовка черновика формулировки;
- подготовка черновика ответа на претензию.

### Обязательные условия для добавления AI

1. Определена модель данных для AI proposals.
2. Есть механизм подтверждения пользователем.
3. AI proposal создаёт audit event.
4. AI не является единственным механизмом для критических операций.
5. Есть тесты для AI-контура.
6. Утверждено владельцем проекта.

### Технологии (будущее)

- LLM API (провайдер определяется отдельно);
- embeddings для поиска по корпусу;
- confidence scoring;
- human-in-the-loop workflow.

### Принцип

**AI — помощник, а не источник истины и не субъект юридического решения.**
